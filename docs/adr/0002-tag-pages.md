# 0002. タグページの追加

Date: 2026-03-15
Status: Accepted

## Context

現在、ブログ記事にはタグが付与されているが（`tags: z.array(z.string()).default([])`）、
PostCard および PostLayout ではタグを `<span class="badge">` として静的に表示しているだけで、
タグによる記事のフィルタリングやナビゲーションができない。

読者がタグをクリックして関連記事を探せるようにするため、以下を追加する。

- タグ一覧ページ (`/tags`) — 全タグとその記事件数を表示
- タグ別記事一覧ページ (`/tags/[tag]`) — 特定タグに紐づく記事を一覧表示
- 既存コンポーネント内のタグ表示をクリック可能なリンクに変更

## Decision

### 実装方針

本機能は Astro の静的ページ生成（`getStaticPaths`）と既存の Content Collections API を活用する。
新しいライブラリの追加は不要。

| 変更種別   | ファイル                        | 概要                                                  |
| ---------- | ------------------------------- | ----------------------------------------------------- |
| 新規ページ | `src/pages/tags/index.astro`    | 全タグ一覧。タグ名と記事件数をバッジで表示            |
| 新規ページ | `src/pages/tags/[tag].astro`    | タグ別記事一覧。`getStaticPaths` で全タグ分を静的生成 |
| 変更       | `src/components/PostCard.astro` | タグの `<span>` を `/tags/[tag]` へのリンクに変更     |
| 変更       | `src/layouts/PostLayout.astro`  | 同上                                                  |
| 変更       | `src/components/Header.astro`   | ナビゲーションに「Tags」リンクを追加                  |

**タグ URL の設計:**

- URL 形式: `/tags/[tag]`（タグ文字列を小文字化してそのまま使用）
- 例: `tags: ["Astro"]` の場合、URL は `/tags/astro`
- タグの正規化は小文字変換（`toLowerCase()`）のみ行う

**タグの集約ロジック:**

- `getCollection("posts")` で全記事を取得し、`draft: true` の記事は除外
- 各記事の `tags` 配列をフラット化し、小文字に正規化して重複を排除
- タグごとの記事件数をカウントし、タグ名のアルファベット順でソート

### 採用しなかった選択肢

| 選択肢                                  | 不採用の理由                                                                             |
| --------------------------------------- | ---------------------------------------------------------------------------------------- |
| タグをスラッグ化（kebab-case 等に変換） | タグは短い英単語が主であり、小文字化のみで十分。過度な変換は可読性を損なう               |
| タグ用の Content Collection を別途定義  | 記事のフロントマターから動的に集約すれば十分。マスターデータの二重管理を避ける           |
| クライアントサイドでのフィルタリング    | SSG のメリットを活かし、タグごとに静的ページを生成する方がパフォーマンス・SEO ともに有利 |

## Consequences

**メリット:**

- 読者がタグ経由で関連記事を発見できるようになる
- 全ページが静的生成されるため、追加のランタイムコストがない
- 既存の BaseLayout / PostCard コンポーネントを再利用するため、UI の一貫性が保たれる

**デメリット・注意点:**

- タグが増えるとビルド時に生成されるページ数が増加する（静的サイトでは一般的なトレードオフ）
- タグの表記揺れ（大文字/小文字の違い）は小文字正規化で吸収するが、同義語の統合は行わない
