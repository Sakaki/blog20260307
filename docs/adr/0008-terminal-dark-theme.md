# 0008. Terminal Dark テーマ

Date: 2026-03-16
Status: Accepted

## Context

現在のブログは daisyUI の `light` テーマをそのまま使用しており、デザイン上の個性が薄い。
技術ブログとしてのブランディングを強化するため、ターミナルを想起させるダークテーマに切り替えたい。

## Decision

- daisyUI の `light` テーマを廃止し、CSS カスタムプロパティベースの独自 "Terminal Dark" テーマを導入する
- フォントは JetBrains Mono（コード・英字）+ Noto Sans JP（日本語）を Google Fonts から読み込む
- 見出しや日付にターミナル風の疑似要素プレフィックスを付与する（`#` `##` `$ date →` など）
- 目次（TOC）のアクティブ状態追跡に Intersection Observer を使用する
- カラーパレットは CSS カスタムプロパティで一元管理する
- MDX コンテンツファイルは変更しない。変更はレイアウト・コンポーネント・スタイルのみ

### 採用しなかった選択肢

1. daisyUI 組み込みの `dark` テーマを使う: ターミナル風の独自表現が難しいため不採用
2. Tailwind のダークモード (`dark:`) で切り替え可能にする: ライト・ダーク切り替え不要の方針のため、ダーク固定とする
3. MDX 内にプレフィックス文字を直接書く: コンテンツとスタイルの分離を維持するため CSS 疑似要素で実装する

## Consequences

### 良い点

- 技術ブログらしい独自のビジュアルアイデンティティが得られる
- CSS カスタムプロパティにより色変更が容易
- MDX コンテンツに手を入れないためコンテンツとスタイルが分離される

### トレードオフ

- Google Fonts への外部依存が増える（オフライン時のフォールバックフォントで対応）
- daisyUI のテーマ機能を活用しなくなるため、daisyUI コンポーネントの一部で色の上書きが必要になる可能性がある
