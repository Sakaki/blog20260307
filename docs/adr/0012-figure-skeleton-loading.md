# ADR 0012: Figure コンポーネントに Skeleton ローディングを追加

## ステータス

承認

## コンテキスト

MDX 記事内で使用する `Figure` コンポーネントで画像を表示する際、画像のロードが完了するまで空白が表示され、ユーザーにとってコンテンツの読み込み状況が分かりにくい。daisyUI の Skeleton コンポーネントを使い、画像ロード中にプレースホルダーを表示することで UX を改善したい。

## 決定事項

### Astro コンポーネントのまま維持し、インラインスクリプトで制御する

React コンポーネント（.tsx）への変換も検討したが、以下の理由から Astro コンポーネント + インラインスクリプトを採用する：

1. **既存の呼び出しへの影響が最小限**: Astro コンポーネントのままなら、MDX 側の import パスや props に変更不要
2. **バンドルサイズの削減**: React ランタイムを画像表示のためだけに読み込む必要がない
3. **実装のシンプルさ**: `<img>` の `onload` イベントと CSS クラスの切り替えだけで実現可能。状態管理ライブラリは不要
4. **SSG との親和性**: Astro の静的生成と相性が良く、ハイドレーション不要

### Skeleton の制御方式

- 初期状態: Skeleton を表示し、`<img>` は `opacity-0` で非表示
- `<img>` の `load` イベント発火時: Skeleton を非表示にし、`<img>` を `opacity-100` に切り替える
- トランジション: `transition-opacity` で滑らかに表示を切り替える
- ブラウザキャッシュ対応: `img.complete` チェックで既にロード済みの画像にも対応する

## 影響範囲

- `src/components/mdx/Figure.astro` -- Skeleton HTML 追加、インラインスクリプト追加
- 既存の MDX 記事の `<Figure>` 呼び出しには変更なし
