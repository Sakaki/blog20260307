# 0007. サイドバーに最近の記事とタグ一覧を追加

Date: 2026-03-16
Status: Accepted

## Context

PC版の記事詳細ページには右ペインに目次（ToC）が表示されているが、目次の下にスペースがあり活用できていない。
読者の回遊性を高めるため、目次の下に「最近の記事」と「タグ一覧」を表示したい。

現状の右ペイン構造:

- `<aside class="hidden lg:block">` の中に `<div class="sticky top-8">` で目次を配置
- 右ペイン幅は `16rem`（256px）

## Decision

- 目次の下に「最近の記事」セクション（最新5件、タイトルと日付のみ）を追加する
- その下に「タグ一覧」セクション（バッジ形式）を追加する
- それぞれ独立した Astro コンポーネント `SidebarRecentPosts.astro` と `SidebarTags.astro` として作成する
- 現在表示中の記事は「最近の記事」から除外する
- sticky コンテナに `max-h-[calc(100vh-4rem)] overflow-y-auto` を適用し、コンテンツが長い場合はサイドバー内でスクロール可能にする

### コンポーネント分離の理由

PostLayout にインラインで書くことも可能だが、以下の理由でコンポーネントに分離する:

1. PostLayout がこれ以上肥大化するのを防ぐ
2. 将来的に他のページ（タグ別記事一覧など）でも再利用できる可能性がある
3. 各セクションのデータ取得ロジックを分離できる

### 採用しなかった選択肢

1. 既存の PostCard コンポーネントを再利用する: サイドバーの幅（16rem）に対して大きすぎるため不適
2. sticky を使わずに固定配置にする: 既存の目次の挙動と統一するため sticky を維持
3. モバイル版にも表示する: 記事本文の下に配置する案もあるが、スコープを絞りPC版のみとする

## Consequences

### 良い点

- 読者の回遊性が向上する
- サイドバーの空きスペースを有効活用できる
- SSG なのでパフォーマンスへの影響はない（ビルド時に全て静的生成）

### トレードオフ

- サイドバーのコンテンツが長くなる場合、スクロール可能領域が必要になる
- 記事数が少ないうちは「最近の記事」の効果が薄い
