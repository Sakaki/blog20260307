# 0003. ハンバーガーメニューによるレスポンシブナビゲーション

Date: 2026-03-15
Status: Accepted

## Context

現在の Header コンポーネントはナビゲーションリンク（Home, Posts, Tags）を横一列に並べている。
デスクトップでは問題ないが、スマートフォンなど横幅の小さい画面ではリンクが窮屈になり、
画面幅が極端に狭い場合はレイアウトが崩れる可能性がある。

モバイルユーザーにとって使いやすいナビゲーションを提供するため、
小さい画面ではハンバーガーメニューにナビゲーションをまとめるレスポンシブ対応を行う。

## Decision

### 実装方針

daisyUI の `dropdown` コンポーネントと Tailwind CSS のレスポンシブクラスを使い、
**JavaScript を一切使わず** HTML/CSS のみでハンバーガーメニューを実現する。

| 変更種別 | ファイル                      | 概要                                                                       |
| -------- | ----------------------------- | -------------------------------------------------------------------------- |
| 変更     | `src/components/Header.astro` | navbar-end 内をレスポンシブ化（モバイル: dropdown / デスクトップ: inline） |

**ブレークポイント設計:**

- モバイル（`< lg`, 1024px 未満）: ハンバーガーアイコンの dropdown メニュー
- デスクトップ（`>= lg`, 1024px 以上）: 従来通りのインラインリンク表示

**使用する daisyUI パターン:**

- `<details>` + `<summary>` による dropdown（CSS の `:focus-within` で開閉、JS 不要）
- `dropdown dropdown-end` で右端揃えのメニューを表示
- `menu` コンポーネントでメニュー項目を構成

### 採用しなかった選択肢

| 選択肢                                | 不採用の理由                                                                  |
| ------------------------------------- | ----------------------------------------------------------------------------- |
| daisyUI drawer コンポーネント         | サイドバー型のナビゲーションは 3 項目程度のメニューには過剰。dropdown で十分  |
| JavaScript によるトグル制御           | SSG の方針に沿い、JS なしで動作する CSS ベースの実装を優先する                |
| 全画面幅でハンバーガーメニューに統一  | デスクトップではインライン表示の方がリンクを直接視認・クリックでき、UX が良い |
| `md`（768px）をブレークポイントにする | タブレット横向きでもインライン表示にしたいため、`lg`（1024px）の方が適切      |

## Consequences

**メリット:**

- モバイルユーザーのナビゲーション体験が改善される
- JavaScript 不要のため、SSG の方針と整合し、バンドルサイズに影響しない
- daisyUI の標準パターンを使うため、テーマとの一貫性が保たれる
- 変更ファイルが Header.astro の 1 ファイルのみで、影響範囲が小さい

**デメリット・注意点:**

- `<details>` ベースの dropdown はメニュー外クリックで閉じるが、ブラウザによって挙動が微妙に異なる場合がある
- ナビゲーション項目が今後大幅に増えた場合、dropdown 内のスクロールを検討する必要がある
