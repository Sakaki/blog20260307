# 0006. 記事ヒーロー画像

Date: 2026-03-16
Status: Accepted

## Context

ブログ記事に視覚的なインパクトを持たせるため、各記事にオプションでヒーロー画像を設定できるようにしたい。
ヒーロー画像は記事詳細ページのヘッダー部分（タグ一覧の下）に表示される。

現状の記事 frontmatter には `title`, `description`, `date`, `tags`, `draft` があるが、画像フィールドはない。

## Decision

- frontmatter に `heroImage`（任意）フィールドを追加する
- `heroImage` は画像 URL（文字列）を受け取る
- 指定がある場合のみ、PostLayout のタグ一覧の下にヒーロー画像を表示する
- 表示には既存の `Figure` コンポーネントのスタイルに合わせて、80%幅・角丸・シャドウ・中央寄せで統一する

### レイヤー構成

このタスクは SSG ブログの UI 機能追加のため、クリーンアーキテクチャのレイヤー構成は適用しない。
変更対象は Astro のスキーマ定義・レイアウト・ページの3箇所。

### 採用しなかった選択肢

1. `heroImage` をオブジェクト型（`{ src, alt, caption }` 等）にする: 現時点ではシンプルな URL で十分。将来拡張が必要になったら対応する
2. `public/images/` にローカル画像を配置する方式: 外部 URL 参照方式で統一する（既存の Figure コンポーネントと同じ方針）

## Consequences

### 良い点

- 記事に視覚的なインパクトを追加できる
- 任意フィールドなので既存記事に影響しない

### トレードオフ

- 外部 URL に依存するためリンク切れのリスクがある
