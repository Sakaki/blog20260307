# Add Reference Images

ブログ記事に、読者の理解を助ける参考画像を追加する。

## 概要

記事で言及されている製品・機器・サービスなどについて、
信頼性の高いソースから参考画像を探し、MDX ファイルに埋め込む。
画像は記事の補足として適度な頻度で配置し、読みやすさを損なわないようにする。

## ワークフロー

### Step 1: 記事の特定と読み込み

ユーザーが指定した記事タイトルまたはスラッグをもとに、
`src/content/posts/` から該当する MDX ファイルを見つけて読み込む。

### Step 2: 画像候補の洗い出し

記事の内容を分析し、画像があると読者の理解が深まるポイントを特定する。

**画像を入れるべき箇所の例**:

- 記事で言及されている具体的な製品・機器（ルーター、スイッチ、キーボードなど）
- ネットワーク機器やハードウェアの外観
- サービスや技術のロゴ・スクリーンショット

**画像が不要な箇所**:

- 概念的な説明や感想パート
- すでに Mermaid 図やコードブロックで視覚的に補足されている箇所
- 導入文や締めの文章

### Step 3: 画像の検索と選定

WebSearch を使って各候補について画像を探す。

**検索のコツ**:

- 製品名 + メーカー名で検索し、公式サイトの製品ページを優先する
- 「〇〇 公式」「〇〇 product page」などで検索するとメーカーページが見つかりやすい
- 製品ページが見つかったら WebFetch でページを確認し、画像 URL を取得する

**画像ソースの優先順位**:

1. メーカー公式サイトの製品画像
2. 公式プレスリリースの画像
3. 大手メディアのレビュー記事の画像

以下のソースは避ける:

- 個人ブログの画像（著作権の問題）
- Amazon等のECサイトの商品画像（利用規約違反の可能性）
- SNS上の画像

**画像の品質基準**:

- 正面からの製品写真や公式のプロモーション画像を優先する
- 解像度が低すぎる画像は避ける
- アスペクト比が極端に横長・縦長の画像は避ける（16:9〜4:3 程度が理想）

### Step 4: MDX への埋め込み

画像は `Figure` コンポーネントを使って埋め込む。

まず、記事の import 文に `Figure` コンポーネントが含まれているか確認し、なければ追加する。

```mdx
import Figure from "../../components/mdx/Figure.astro";
```

画像の埋め込みには以下の形式を使う。

```mdx
<Figure
  src="https://example.com/image.jpg"
  alt="製品名の説明"
  sourceName="メーカー名"
  sourceUrl="https://example.com/product-page"
/>
```

**props**:

- `src`（必須）: 画像の URL
- `alt`（必須）: 製品名と簡潔な説明（例: `MikroTik CRS305-1G-4S+in の外観`）
- `sourceName`（必須）: 出典元の名前（例: `MikroTik`）
- `sourceUrl`（必須）: 出典元のページ URL
- `caption`（任意）: 画像の上に表示するキャプション。省略時は `alt` の値が使われる

**配置ルール**:

- h2 セクションごとに 0〜2 枚。内容や文章量に応じて判断する
  - 具体的な製品・機器に言及しているセクション → 1〜2 枚
  - 感想や概念的な内容のセクション → 0 枚でよい
  - すでに Mermaid 図がある箇所 → 0〜1 枚に抑える
- 画像は関連する文章の直後、次の見出しの前に配置する
- 記事全体で画像が偏らないようバランスを取る
- 迷ったら「少なめ」を選ぶ。読みやすさが最優先

### Step 5: 確認と報告

画像を追加した後、以下を報告する:

- 追加した画像の一覧（製品名、配置箇所、出典URL）
- 追加しなかった h2 セクションとその理由（もしあれば）

画像が見つからなかった製品については、その旨を報告し、
ユーザーに代替案を提案する（例: 類似製品の画像、スクリーンショットの撮影依頼など）。

## 注意事項

- 画像の外部 URL は将来リンク切れになる可能性がある。
  公式サイトの製品ページ画像は比較的安定しているため優先する。
- 著作権に配慮し、出典を必ず明記する。
- 記事の読みやすさが最優先。画像が多すぎてごちゃつくくらいなら少なめにする。
