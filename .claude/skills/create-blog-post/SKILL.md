# create-blog-post

このブログ（Astro + MDX）の記事を作成して `src/content/posts/` に配置するワークフロー。

## ステップ 1: ヒアリング

記事作成の前に、以下をユーザーに確認する。まだ決まっていない項目だけ聞けばよい。

1. **記事のテーマ・タイトル候補** — 何について書きたいか
2. **書きたい内容** — 箇条書きでざっくりリストアップしてもらう（順序不問）
3. **タグ** — `keyboard`, `gear`, `review`, `astro`, `tips` など（なければ省略可）
4. **公開日** — 未指定なら今日の日付を使う

ヒアリングは一度にまとめて聞くこと。何度も往復しない。

## ステップ 2: 既存記事のスタイル確認（毎回必須）

**スキル実行のたびに** `src/content/posts/` 内の既存記事を最新5件まで読み、その時点の文体・スタイルを把握する。
（`date` フィールドが新しい順に最大5件。記事は随時更新されるため、ファイルを実際に読んで最新の傾向を確認すること。固定のルールに頼らない。）

確認する観点：

- **語尾・文体**: です・ます体か、だ・である体か、混在しているか
- **書き出しのパターン**: どう始まっているか（セクション有無、冒頭の文章の雰囲気）
- **セクション構成**: `##` の粒度、まとめセクションの有無
- **強調・リストの使い方**: 太字の使い方、箇条書きのスタイル
- **全体のトーン**: 口語寄りか、硬めか

新しい記事はここで把握したスタイルに合わせて書く。

## ステップ 3: 記事の構成を組み立てる

ユーザーから受け取った箇条書きリストを分析し、読み手にとって自然な流れに並べ直す。

構成の考え方：

- **冒頭**: 何について書くのか、なぜ書くのかを1〜2文で示す
- **中盤**: 詳細・経緯・比較・理由など
- **末尾**: 感想・まとめ・今後の展望など（自然に終わればよい）

ユーザーのメモに書いてある情報だけを使う。勝手に事実を補完・創作しない。
情報が不足している箇所は「〜は後日追記予定」など曖昧にせず、その項目を省くか、
執筆前にユーザーに確認する。

## ステップ 4: フロントマターの生成

```yaml
---
title: "記事タイトル"
description: "1〜2文の要約。PostCard に表示される。"
date: YYYY-MM-DD
tags: ["tag1", "tag2"]
draft: false
---
```

- `slug`（ファイル名）は英数字・ハイフン区切り（例: `my-keyboard-review`）
- `draft: true` にすると公開されないので、完成した記事は `draft: false`
- `description` はSEOと一覧表示に使われるため、内容を端的に表す

## ステップ 5: MDX ファイルの作成

`src/content/posts/<slug>.mdx` として保存する。

カスタムコンポーネントが必要な場合のインポート例：

```mdx
import Callout from "../../components/mdx/Callout.tsx";
import CodeBlock from "../../components/mdx/CodeBlock.tsx";
```

通常の記事では不要なことが多い。コードサンプルや注意書きが必要な場合のみ使う。

## ステップ 6: 読みやすさ・面白さのセルフレビューと改善

MDX ファイルを保存したあと、必ず読み返して以下の観点で品質を確認し、問題があれば即座に修正する。ユーザーへの確認は不要。

**構成の観点**

- 薄いセクション（2文以下で終わっているなど）は、冒頭や隣接する段落に吸収できないか
- セクションの順序は読者にとって自然な流れか（背景 → 詳細 → 感想 の流れが崩れていないか）

**文章の観点**

- 前後の文・段落との繋がりが唐突に切れていないか
- 結論や感想が宙に浮いていないか（前の文脈から自然に導かれているか）
- 同じ語尾・表現が続いていないか

**面白さの観点**

- 書き手の視点・感想・判断が伝わるか（事実の羅列になっていないか）
- 読者が「なるほど」「わかる」と思えるような具体性や文脈があるか

修正後、変更した箇所と理由をユーザーに簡潔に報告する。

## 注意事項

- `draft: true` の記事はビルドから除外されるため、意図しない限り `draft: false` にする
- ファイル名（slug）に日本語・スペースは使わない
- 記事 URL は `/posts/<slug>` になる
